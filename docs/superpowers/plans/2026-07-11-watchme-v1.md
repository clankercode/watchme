# WatchMe v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a production-quality local supervisor that is registered with bare `!watchme`, safely detects and recovers blocked coding-agent sessions, and exits when the watched agent exits.

**Architecture:** One Rust 2024 package installs one `watchme` binary plus a `WatchMe` compatibility link. Focused modules implement secure local IPC and persistence, exact process/pane identity, tmux and Herdr adapters, normalized observation and policy, Claude/Codex recovery, manifest-driven generic agents, and a constrained alternate-provider planner.

**Tech Stack:** Rust 2024, Tokio current-thread runtime, Clap, Serde/JSON/TOML, JSON Schema, tracing, chrono/chrono-tz, regex-automata, nix/libc, tempfile, proptest, Unix sockets, tmux, and schema-faithful fixture services.

---

## File map

- `Cargo.toml`, `Cargo.lock`: package metadata, `Unlicense OR CC0-1.0`, dependencies, lint policy.
- `src/main.rs`, `src/cli.rs`: bare registration and administrative command dispatch.
- `src/config.rs`, `src/paths.rs`: typed configuration and secure XDG paths.
- `src/model/*`: identities, normalized events, actions, watcher state.
- `src/ipc/*`, `src/daemon/*`, `src/store.rs`: owner-only protocol, scheduler, registry, atomic state.
- `src/process/*`, `src/mux/*`: process ancestry/identity and tmux/Herdr operations.
- `src/observe/*`, `src/policy.rs`, `src/recovery/*`: signal collection, sanitizing, classification, state machine, safe actions.
- `src/agents/*`: Claude, Codex, and manifest-based adapters.
- `src/planner/*`, `src/redact.rs`: provider-independent constrained planner and snapshots.
- `src/notify.rs`, `src/doctor.rs`: notifications and actionable diagnostics.
- `schemas/`, `config/`, `manifests/`, `fixtures/`: runtime contracts and compatibility data copied from the bundle.
- `tests/`: CLI, daemon, tmux, Herdr, recovery, hostile-input, crash, and lifecycle integration suites.
- `scripts/`, `packaging/`: install/uninstall, hooks, services, resource benchmark, validation.
- `docs/`: command/config/manifest/security/compatibility/benchmark/release documentation.
